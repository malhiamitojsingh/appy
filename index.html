how it works

Three steps, then done.

1

Download & run

Get appy.sh and run it as root. No installation, no dependencies, no config files.

2

Pick your services

Numbered menu appears. Type the numbers you want, space-separated. Type A for everything, M for maintenance.

3

Services are running

Each service installs via pacman or yay, then gets enabled with systemd immediately. No reboot needed.

curl -fsSL https://YOUR-PROJECT.pages.dev/appy.sh -o appy.sh && sudo bash appy.sh

deploy to cloudflare pages

Host appy.sh free in 5 minutes.

Cloudflare Pages is free, global CDN, automatic HTTPS. Your script stays always-available.

1

Create a GitHub repo

Go to github.com/new. Name it appy. Set to Public. Click Create repository.

2

Push your files

Add appy.sh and index.html to the repo root and push.

git clone https://github.com/YOUR_USERNAME/appy.git && cd appy
git add appy.sh index.html && git commit -m "init" && git push
3

Connect to Cloudflare Pages

Go to dash.cloudflare.com → Workers & Pages → Create → Pages → Connect to Git. Select your appy repo.

4

Build settings — leave everything blank

Framework presetNone Build command(empty) Build output directory(empty) Root directory(empty)

Click Save and Deploy. Wait ~30 seconds.

5

Update the download URL

You'll get a URL like appy-abc123.pages.dev. Replace YOUR-PROJECT.pages.dev in index.html with your real URL, then push. Cloudflare auto-redeploys.

curl -fsSL https://appy-abc123.pages.dev/appy.sh -o appy.sh && sudo bash appy.sh
6

Custom domain (optional)

In Cloudflare Pages → your project → Custom domains → Set up a custom domain. If your domain is on Cloudflare, it connects in one click.

maintenance

Keep your system healthy.

Re-run sudo bash appy.sh anytime and press M for the maintenance menu.

Update all packages

Full system update via pacman including all installed packages.

pacman -Syu

Update AUR packages

Updates AUR-installed packages via yay.

yay -Syu

Clean package cache

Removes cached package files to free up disk space.

pacman -Sc

Check failed services

Shows any systemd services that failed to start properly.

systemctl --failed

Disk usage

Shows disk usage for root filesystem and package cache.

df -h /

Running services

Lists all currently active systemd services.

systemctl list-units --state=running

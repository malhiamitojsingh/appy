how it works

Four steps, then it manages itself.

1

Download & run

Get appy.sh and run as root. No install, no deps, no config files.

2

Pick services

Grouped menu. Type numbers space-separated. A for everything, H for health.

3

Services running

Each service installs via pacman, yay, or docker. Enabled with systemd. No reboot needed.

4

Daemon watches

Install the watchdog from M → 7. It auto-restarts failed services and alerts you of issues.

curl -fsSL https://YOUR-PROJECT.pages.dev/appy.sh -o appy.sh && sudo bash appy.sh
new in v2.0

Beyond installing — a full system manager.

appy now monitors your system continuously, restarts failed services, schedules updates, and keeps your configs backed up.

🐕

Watchdog Daemon

Runs as a systemd service. Checks every 5 minutes, auto-restarts crashed services, and logs every event.

📊

Real-time Health Check

CPU, memory, disk, load average, network interfaces, and per-service status in one color-coded report.

🔄

Auto-Update Scheduler

systemd timer runs pacman, yay, and Docker image pulls automatically. Daily or weekly — your choice.

💾

Config Backup

Archives configs for Nginx, Caddy, Prometheus, Grafana, UFW, WireGuard, and more. Keeps last 5 archives.

↩

Service Rollback

Remove any installed service cleanly — stops, disables, and uninstalls. Docker containers included.

🔔

Notifications & Alerts

Disk >90%, memory >95%, service crashes, and pending updates logged and optionally emailed.

watchdog daemon

Set it and forget it.

Install the appy daemon once and it monitors your services continuously — restarting failures, logging health, and alerting on problems.

🐾

Service Watchdog

Polls all appy-managed services every 5 minutes. Restarts any that have crashed and logs the event.

📈

Resource Alerts

Warns when disk exceeds 90% or memory exceeds 95%. All alerts logged to a notifications file.

📦

Update Detection

Checks for available package updates every 6 hours. Shows pending count in the main menu.

📜

Log Rotation

Automatically rotates appy logs when they exceed 10 MB. Keeps last 3 backups.

📧

Email Notifications

Set APPY_NOTIFY_EMAIL to receive alerts via email when services fail.

# Install via maintenance menu
sudo bash appy.sh
→ M → 7

# Or install directly
sudo bash appy.sh --daemon &

# Check daemon status
systemctl status appy-daemon

# Watch live logs
tail -f /var/log/appy-daemon.log

# Configure check interval (seconds)
APPY_DAEMON_INTERVAL=120
# Email alerts (requires mail)
APPY_NOTIFY_EMAIL="you@example.com"

deploy to cloudflare pages

Host appy.sh free in 5 minutes.

Cloudflare Pages — free, global CDN, automatic HTTPS. Your script stays always-available.

1

Create a GitHub repo

Go to github.com/new. Name it appy. Set to Public. Click Create repository.

2

Push your files

Add appy.sh, index.html, and README.md to the repo root and push.

git clone https://github.com/YOUR_USERNAME/appy.git && cd appy
git add . && git commit -m "appy v2.0" && git push
3

Connect to Cloudflare Pages

Go to dash.cloudflare.com → Workers & Pages → Create → Pages → Connect to Git. Select your appy repo.

4

Build settings — leave everything blank

Framework presetNone Build command(empty) Build output directory(empty) Root directory(empty)

Click Save and Deploy. Wait ~30 seconds.

5

Update the download URL

You'll get a URL like appy-abc123.pages.dev. Replace YOUR-PROJECT.pages.dev everywhere in the files, then push. Cloudflare auto-redeploys.

curl -fsSL https://appy-abc123.pages.dev/appy.sh -o appy.sh && sudo bash appy.sh
6

Custom domain (optional)

Cloudflare Pages → your project → Custom domains → Set up a custom domain. If your domain is on Cloudflare, it connects in one click.

Config Backup

sudo bash appy.sh --backup

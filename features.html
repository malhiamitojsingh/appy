01
hardware intelligence

Knows your machine before it touches it.

Before installing a single package, the script reads everything about your server — CPU cores, available RAM, disk type (NVMe / SSD / HDD), GPU model, and battery. Then it decides what's safe.

  • CPU model, cores, threads, architecture (x86_64 / ARM)
  • RAM total and available, swap size and current usage
  • Disk type via sysfs rotational flag
  • GPU via lspci for AI and media server recommendations
  • Battery detected → TLP auto-selected for laptop power management

02
rule engine

Blocks mistakes before you make them.

Every component has minimum requirements — RAM, CPU cores, disk space, and whether it tolerates spinning rust. The engine evaluates each one against your hardware and gates it. No accidental k8s on 512 MB RAM.

  • LOW: CLI tools only — DBs and Docker blocked
  • MEDIUM: Docker + single container, warns on MySQL/PostgreSQL
  • HIGH: full stack including Kubernetes and media servers
  • HDD detected → warns before any database or transcoding install
  • All gates overridable with --force if you're sure

03
execution order

Installs in the right order. Every time.

A 7-phase state machine ensures base hardening always runs first, containers before the apps that need them, databases before web apps that depend on a backend.

  • Phase 1: Pre-flight checks and config backups
  • Phase 2: Base hardening — UFW, SSH, fail2ban, etckeeper
  • Phase 3: Container runtimes — Docker, Podman
  • Phase 4: Databases — before anything that depends on them
  • Phases 5–7: Web servers, services, post-install hooks

04
safety & rollback

Run it twice. Nothing explodes.

State files in /var/lib/bashonacci/ track what's already done. etckeeper snapshots /etc before every change. Roll back any single component if something goes wrong.

  • State files prevent duplicate installs across runs
  • etckeeper git-commits /etc before each component
  • --rollback <name> purges any single component cleanly
  • Existing configs never overwritten without --force
  • Full log at /var/log/ plus timestamped copy in ~/

also

More things it does.

The smaller details that still matter.

⚑

Port conflict detection

Checks 80, 443, 3306, 5432 before installing anything that needs them. Skips with a warning, not a crash.

🔐

Credential logging

Auto-generated passwords are written to a timestamped log in ~/ — never lost, never printed to stdout where they'd scroll past.

🔒

Single-instance lock

Creates a PID lock file so two concurrent runs can't fight over the same system. Second one exits cleanly.

🧱

Non-Ubuntu abort

Reads /etc/os-release and hard-aborts if it's not Ubuntu ≥ 20.04. No silent breakage on Debian or CentOS.

⚡

Laptop mode

Battery detected → TLP automatically added. Suspend and power-save tuning without asking.

🧩

YAML-extensible

Adding a new component means editing exactly one array. The install loop, rule engine, and rollback all work automatically.
